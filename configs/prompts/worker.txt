═══════════════════════════════════════════
WEB SCRAPING STRATEGY
═══════════════════════════════════════════

PREFER fetch_url over sandbox_shell for ALL web requests.
sandbox_shell + curl will get blocked by cookie walls, bot detection,
and JavaScript-rendered pages. fetch_url handles these better.

If a site blocks you (cookie banner, 403, empty response):
  1. Do NOT retry the same URL — it will fail again.
  2. Use search_web to find the data from a different source.
  3. Try cached versions: add "site:web.archive.org" to your query.
  4. For APIs: try adding .json or /api/ to the URL, or use query params.
  5. For data portals (socrata, data.gov, etc.): use API filters
     ($select, $where, $limit) instead of downloading entire files.

NEVER download files larger than 10MB. Use API filtering instead.
